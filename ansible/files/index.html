Stack technique

            
Technologies utilisées

            

                Une stack open source et cloud éprouvée, choisie pour couvrir
                tout le cycle de vie de l'infrastructure : du commit au
                serveur en production.
            


            

                

                    
☁

                    
AWS

                    
Cloud public : EC2, VPC, Security Groups pour la partie hébergée du lab.

                

                

                    
⬡

                    
Terraform

                    
Infrastructure as Code déclarative : l'infra décrite en HCL, reproductible à l'identique.

                

                

                    
A

                    
Ansible

                    
Gestion de configuration sans agent : playbooks idempotents, inventaires hybrides.

                

                

                    
⚙

                    
Jenkins

                    
Serveur CI/CD : pipelines déclaratifs (Jenkinsfile), intégration Git et notifications.

                

                

                    
◧

                    
Docker

                    
Conteneurisation des outils et environnements de test isolés et jetables.

                

                

                    
🐧

                    
Linux

                    
Système cible (Ubuntu/Debian) : durcissement, services réseau, supervision.

                

                

                    
⎇

                    
Git

                    
Source unique de vérité : branches, merge requests, traçabilité complète des changements.

                

                

                    
🛡

                    
Security

                    
Sécurité transverse : scans automatisés, moindre privilège, secrets chiffrés.

                

            

        

    


    
    

        

            
Shift-left security

            
La sécurité intégrée au pipeline

            

                Plutôt que d'auditer l'infrastructure après coup, chaque contrôle
                de sécurité est exécuté automatiquement avant le
                déploiement. Un code non conforme ne quitte jamais le pipeline.
            


            

                

                    
✓

                    

                        
Infrastructure as Code (IaC)

                        
L'infrastructure entière est décrite en code versionné : auditable, revue par les pairs, reconstruite à l'identique en cas d'incident.

                    

                    terraform / ansible
                

                

                    
✓

                    

                        
CI/CD sécurisé

                        
Pipeline Jenkins avec étapes bloquantes : aucun déploiement si un scan échoue. Secrets gérés hors du code (credentials Jenkins).

                    

                    jenkinsfile
                

                

                    
✓

                    

                        
tfsec

                        
Analyse statique du code Terraform : détecte les Security Groups trop ouverts, le chiffrement manquant, les mauvaises pratiques AWS.

                    

                    tfsec .
                

                

                    
✓

                    

                        
ansible-lint

                        
Vérification des playbooks Ansible : idempotence, bonnes pratiques, permissions de fichiers, modules dépréciés.

                    

                    ansible-lint playbooks/
                

                

                    
✓

                    

                        
Pré-commit hooks

                        
Contrôles exécutés sur le poste du développeur avant même le commit : formatage, détection de secrets, validation syntaxique.

                    

                    pre-commit run

Objectifs du mémoire

                    

                        Démontrer qu'une équipe réseau peut adopter les pratiques
                        DevSecOps pour fiabiliser et sécuriser ses opérations,
                        avec des outils open source accessibles.
                    

                    
    
                        
  • Réduire les erreurs manuelles grâce à l'automatisation de bout en bout.
    • 
                        
  • Garantir la conformité sécurité par des contrôles automatisés et bloquants.
    • 
                        
  • Unifier la gestion du cloud (AWS) et du réseau local dans un même pipeline.
    • 
                        
  • Assurer la traçabilité complète : qui a changé quoi, quand, et pourquoi.
    •
